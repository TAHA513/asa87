يبدو أنك تواجه مشكلتين:

خطأ EADDRINUSE (المنفذ قيد الاستخدام):

هذا الخطأ يعني أن المنفذ 5000 الذي تحاول تشغيل الخادم عليه مستخدم بالفعل من قبل تطبيق آخر.

الحلول:

إعادة تشغيل التطبيق: في Replit، يمكنك إيقاف التطبيق ثم إعادة تشغيله. غالبًا ما يحل هذا المشكلة.

تغيير المنفذ: إذا استمرت المشكلة، يمكنك تغيير المنفذ في الكود الخاص بك إلى منفذ آخر غير مستخدم، مثل 5001 أو 8080. على سبيل المثال:

javascript
Copy
app.listen(5001, '0.0.0.0', () => {
  console.log('Server is running on port 5001');
});
إيقاف التطبيق الذي يستخدم المنفذ: إذا كنت تعمل على جهاز محلي (وليس Replit)، يمكنك استخدام الأوامر التالية لإيجاد وإيقاف العملية التي تستخدم المنفذ:

bash
Copy
sudo lsof -i :5000
sudo kill -9 <PID>
حيث <PID> هو معرف العملية.

خطأ AssertError: Expected union value:

هذا الخطأ يشير إلى أن القيمة 'modern' التي تم تمريرها للحقل /variant غير مقبولة وفقًا للـ schema المحدد.

الحلول:

تحقق من الـ schema: راجع الـ schema الخاص بالحقل /variant وتأكد من القيم المسموح بها. على سبيل المثال، إذا كان الـ schema يتوقع قيمًا مثل 'default' أو 'dark' أو 'light'، فإن 'modern' ليست قيمة صالحة.

تعديل القيمة: قم بتغيير القيمة 'modern' إلى واحدة من القيم المسموح بها في الـ schema. على سبيل المثال:

javascript
Copy
const theme = {
  variant: 'default' // أو 'dark' أو 'light' حسب الـ schema
};
تعديل الـ schema: إذا كنت تريد أن تكون 'modern' قيمة مقبولة، يمكنك تعديل الـ schema ليشمل هذه القيمة. على سبيل المثال:

javascript
Copy
const schema = Type.Union([
  Type.Literal('default'),
  Type.Literal('dark'),
  Type.Literal('light'),
  Type.Literal('modern')
]);
ملاحظة إضافية:
إذا كنت تستخدم Replit وتواجه مشكلة في فتح المنفذ 5000 بعد تشغيل الأمر Start application، فقد يكون ذلك بسبب تأخير في تهيئة البيئة. يمكنك تجربة ما يلي:

إعادة تشغيل Replit: قم بإيقاف التطبيق ثم إعادة تشغيله.

زيادة وقت الانتظار: إذا كان هناك خيار لزيادة وقت الانتظار قبل فتح المنفذ، قم بزيادته إلى أكثر من 60000ms.

إذا استمرت المشكلة، قد تحتاج إلى التحقق من إعدادات Replit أو الاتصال بدعمهم للحصول على مساعدة إضافية.
