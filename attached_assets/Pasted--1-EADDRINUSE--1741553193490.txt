
يبدو أن لديك مشكلتين رئيسيتين في الكود الذي تعمل عليه:

1. خطأ EADDRINUSE (المنفذ قيد الاستخدام):
الخطأ: code: 'EADDRINUSE', errno: -98, syscall: 'listen', address: '0.0.0.0', port: 5000

التفسير: هذا الخطأ يعني أن المنفذ 5000 على العنوان 0.0.0.0 قيد الاستخدام بالفعل من قبل تطبيق آخر. لا يمكن لتطبيقين أن يستخدما نفس المنفذ في نفس الوقت.

الحلول:

تغيير المنفذ: يمكنك تغيير المنفذ في الكود الخاص بك إلى منفذ آخر غير مستخدم، مثل 5001 أو 8080.

javascript
Copy
app.listen(5001, '0.0.0.0', () => {
  console.log('Server is running on port 5001');
});
إيقاف التطبيق الذي يستخدم المنفذ: يمكنك إيجاد التطبيق الذي يستخدم المنفذ 5000 وإيقافه. على سبيل المثال، في نظام Linux، يمكنك استخدام الأمر التالي لإيجاد العملية التي تستخدم المنفذ:

bash
Copy
sudo lsof -i :5000
ثم يمكنك إيقاف العملية باستخدام:

bash
Copy
sudo kill -9 <PID>
حيث <PID> هو معرف العملية الذي حصلت عليه من الأمر السابق.

2. خطأ AssertError: Expected union value:
الخطأ: AssertError: Expected union value مع تفاصيل إضافية تشير إلى أن القيمة 'modern' غير مقبولة للحقل /variant.

التفسير: هذا الخطأ يشير إلى أن القيمة 'modern' التي تم تمريرها للحقل /variant لا تتوافق مع القيم المتوقعة في الـ schema. الـ schema يتوقع قيمة من نوع union (أي قيمة من مجموعة محددة من القيم)، ولكن 'modern' ليست واحدة من هذه القيم.

الحلول:

تحقق من الـ schema: قم بمراجعة الـ schema الذي تم تعريفه للحقل /variant وتأكد من القيم المسموح بها. على سبيل المثال، إذا كان الـ schema يتوقع قيمًا مثل 'default' أو 'dark' أو 'light'، فإن 'modern' ليست قيمة صالحة.

تعديل القيمة: قم بتغيير القيمة 'modern' إلى واحدة من القيم المسموح بها في الـ schema. على سبيل المثال:

javascript
Copy
const theme = {
  variant: 'default' // أو 'dark' أو 'light' حسب الـ schema
};
تعديل الـ schema: إذا كنت تريد أن تكون 'modern' قيمة مقبولة، يمكنك تعديل الـ schema ليشمل هذه القيمة. على سبيل المثال:

javascript
Copy
const schema = Type.Union([
  Type.Literal('default'),
  Type.Literal('dark'),
  Type.Literal('light'),
  Type.Literal('modern')
]);
ملخص:
بالنسبة لخطأ EADDRINUSE، قم بتغيير المنفذ أو إيقاف التطبيق الذي يستخدم المنفذ.

بالنسبة لخطأ AssertError، تحقق من الـ schema وقم بتعديل القيمة أو الـ schema وفقًا لذلك.
